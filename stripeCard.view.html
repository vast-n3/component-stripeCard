Stripe-Card

<stripe-card :card-holder="name" @card-token="cardToken = $event"></stripe-card> see demo.js

Accepts credit/debit card inputs, verifies input & emits a one-time token-id to be used for on the server.
This element requires server-side passed in values for "stripePublicKey":

...
->vueComponent('stripeCard',['stripePublicKey' => 'pk_test_123456787'])
-> ...

Also, make sure to include the stripe.js script either in your frame or on the route-component.

Prop Type Function
cardHolder String (cardholder full name) Optional, but recommended
country String (2char representation e.g. 'US') Optional, but recommended
currency String (e.g. 'usd') Note: this is only necessary for Stripe Connect and will otherwise be ignored

This element can be (and should be) placed within a component form. Property bindings are two-way. We suggest passing in the full cardholder name as well as the country. Stripe RADAR can then better prevent fraudulent transactions.

About security: Credit card information is never sent to your server with this approach. The provided token can be used to request a server-side charge against the payment method using the Stripe model.
